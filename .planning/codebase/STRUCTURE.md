## Directory Purposes

**`src/pages/`:**
- Purpose: One file per route. Each file is a full-page React component.
- Contains: 15 page files + `NotFound.tsx`
- Key files:
  - `src/pages/Index.tsx` — unified login with médico/paciente toggle
  - `src/pages/Cadastro.tsx` — user registration
  - `src/pages/DashboardMedico.tsx` — médico home, uses `DashboardLayout` with `navItems`
  - `src/pages/DashboardPaciente.tsx` — paciente home
  - `src/pages/PrescricaoDigital.tsx` — prescription management
  - `src/pages/Anamnese.tsx` — clinical history form
  - `src/pages/Agenda.tsx` — appointment calendar (médico)
  - `src/pages/PainelMonitoramento.tsx` — patient monitoring panel
  - `src/pages/PainelIA.tsx` — AI panel
  - `src/pages/MeuProntuario.tsx` — patient's own medical record
  - `src/pages/AgendaPaciente.tsx` — appointment booking (paciente)
  - `src/pages/ReceitasPaciente.tsx` — patient prescriptions view
  - `src/pages/Especialistas.tsx` — doctor search

**`src/components/`:**
- Purpose: Shared layout components and route guard. NOT domain components.
- `AuthLayout.tsx` — used by `Index`, `Cadastro`, `EsqueciSenha`
- `DashboardLayout.tsx` — used by all `/medico/*` and `/paciente/*` pages; receives `navItems: NavItem[]` prop
- `NavLink.tsx` — wraps React Router `NavLink` with `activeClassName` support
- `ProtectedRoute.tsx` — auth guard; accepts `requiredTipo="medico"|"paciente"`

**`src/components/ui/`:**
- Purpose: shadcn/ui generated primitives — accordion, button, card, dialog, form, sidebar, toast, etc.
- Generated by: `npx shadcn@latest add [component-name]`
- Rule: NEVER edit manually. Re-generate via CLI to update.

**`src/contexts/`:**
- Purpose: React Context providers for cross-cutting state
- `AuthContext.tsx` — only context in the app; exposes `useAuth()` hook

**`src/hooks/`:**
- Purpose: All Supabase data access. Each file groups queries/mutations for one domain entity.
- `useConsultas.ts` — `useConsultasMedico`, `useConsultasPaciente`, `useProximaConsultaMedico`, `useProximaConsultaPaciente`, `useCreateConsulta`, `useUpdateConsultaStatus`
- `usePacientes.ts` — `useMeusPacientes`, `usePacienteByConsultas`, `useTodosMedicos`
- `usePrescricoes.ts` — `usePrescricoesMedico`, `usePrescricoesPaciente`, `useCreatePrescricao`, `useUpdatePrescricao`
- `useAnamneses.ts` — `useAnamnesesMedico`, `useAnamnesesPaciente`, `useSalvarAnamnese`
- `use-toast.ts` — shadcn/ui toast utility hook
- `use-mobile.tsx` — responsive breakpoint hook

**`src/lib/`:**
- Purpose: Shared infrastructure utilities
- `supabase.ts` — Supabase JS client singleton (reads `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY`)
- `database.types.ts` — TypeScript interfaces for all domain entities: `Profile`, `Medico`, `Paciente`, `Consulta`, `Prescricao`, `Anamnese`, `Notificacao`, `MedicamentoItem`
- `utils.ts` — `cn()` helper (`clsx` + `tailwind-merge`)
- `errors.ts` — error normalization utilities
- `masks.ts` — CPF/CRM input mask formatters

**`src/test/`:**
- Purpose: Vitest unit tests and test configuration
- `setup.ts` — Vitest global setup
- `masks.test.ts` — tests for `src/lib/masks.ts`
- `errors.test.ts` — tests for `src/lib/errors.ts`
- `example.test.ts` — example test scaffolding

**`supabase/migrations/`:**
- Purpose: Versioned SQL migrations for the Supabase Postgres schema
- `001_initial_schema.sql` — tables: profiles, medicos, pacientes, consultas, prescricoes, anamneses, notificacoes + RLS + triggers
- `002_fix_policies.sql` — RLS policy corrections
- `003_backfill_and_robust.sql` — data backfills

## Naming Conventions

**Files:**
- Pages: `PascalCase.tsx` matching route concept — `DashboardMedico.tsx`, `PrescricaoDigital.tsx`
- Hooks: `camelCase.ts` prefixed with `use` — `useConsultas.ts`, `usePacientes.ts`
- Components: `PascalCase.tsx` — `DashboardLayout.tsx`, `ProtectedRoute.tsx`
- Lib utilities: `camelCase.ts` — `utils.ts`, `masks.ts`, `errors.ts`
- Tests: co-located in `src/test/`, named `<subject>.test.ts`

**React Exports:**
- Pages and layouts: `export default function ComponentName()`
- Hooks: named exports — `export function useHookName()`
- Context: named export for provider + named export for hook — `export function AuthProvider()`, `export function useAuth()`

**Variables / Functions:**
- All in Portuguese (pt-BR) following domain language — `medico`, `paciente`, `consulta`, `prescricao`, `anamnese`
- Query keys: lowercase kebab-style string arrays — `["consultas", "medico", userId]`

## Component Organization Patterns

**Page Component Structure:**
Each page imports `DashboardLayout` (for authenticated) or `AuthLayout` (for public), defines its own `navItems` array if it's a top-level dashboard, and calls the relevant domain hooks.

```typescript
// Pattern: authenticated page
export default function PageName() {
  const { profile, medico } = useAuth();
  const { data = [] } = useDomainHook();
  return (
    <DashboardLayout navItems={navItems} >
      {/* page content */}
    </DashboardLayout>
  );
}
```

**Hook Structure:**
Each hook file exports multiple named functions grouped by entity. Read hooks use `useQuery`, write hooks use `useMutation`.

```typescript
// Pattern: read hook
export function useEntityRole() {
  const { user } = useAuth();
  return useQuery({
    queryKey: ["entity", "role", user?.id],
    enabled: !!user?.id,
    queryFn: async () => { /* supabase query */ },
  });
}

// Pattern: write hook
export function useCreateEntity() {
  const { user } = useAuth();
  const qc = useQueryClient();
  return useMutation({
    mutationFn: async (data) => { /* supabase insert */ },
    onSuccess: () => qc.invalidateQueries({ queryKey: ["entity"] }),
  });
}
```

## Where to Add New Code

**New authenticated page (médico):**
1. Create `src/pages/NovaFuncionalidade.tsx` using `DashboardLayout`
2. Add route in `src/App.tsx`: `<Route path="/medico/nova-funcionalidade" element={<ProtectedRoute requiredTipo="medico"><NovaFuncionalidade /></ProtectedRoute>} />`
3. Add entry to `navItems` array in `src/pages/DashboardMedico.tsx` if it should appear in sidebar

**New authenticated page (paciente):**
1. Same pattern under `src/pages/` and `/paciente/` prefix
2. Add `navItems` entry in `src/pages/DashboardPaciente.tsx`

**New data hook (new entity or new query on existing entity):**
- Add to the relevant existing file in `src/hooks/` (e.g., new consulta query → `src/hooks/useConsultas.ts`)
- If entity is new, create `src/hooks/useNomeEntidade.ts`
- Always: `enabled: !!user?.id`, scoped query key with `user?.id`

**New domain type:**
- Add interface to `src/lib/database.types.ts`

**New UI primitive (shadcn/ui):**
- Run `npx shadcn@latest add [component-name]`
- Output goes to `src/components/ui/` automatically

**New shared utility:**
- Add to `src/lib/utils.ts` (generic) or create `src/lib/newUtil.ts` (domain-specific)

**New database migration:**
- Add numbered file `supabase/migrations/006_description.sql`
- Apply with `supabase db push`
